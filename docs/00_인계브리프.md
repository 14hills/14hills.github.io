# 14hills 운영 앱 — SaiZen 프로젝트 인계 브리프

> 이 문서를 들고 **SaiZen 프로젝트**로 이동해 14hills 운영 앱을 완성하세요.
> SaiZen 앱 v13의 구조·코드 패턴이 그쪽에 있어, 거기서 작업하는 것이 가장 빠릅니다.

---

## 1. 무엇을

SaiZen 야마나미 운영 자동화 앱의 구조를 그대로 차용해
**포틴힐즈(14hills) 전용 운영 도구**를 만든다.

## 2. 대상

- 포틴힐즈 컨트리클럽 (나고야 名古屋) · **18홀** 골프장
- DB: **14hills 전용 Supabase** (SaiZen DB와 완전 분리 — 키·RLS 별도)

## 3. 복제할 모듈 (4종)

SaiZen v13(7탭) 중 아래 4개만 가져와 14hills용으로 분기.

| 모듈 | SaiZen 대응 | 역할 |
|---|---|---|
| 네임택 | 네임택 | 참가자 네임택 출력 (그룹/출발지 기준) |
| 현지 수배서 | 송영·숙박·골프 통합 | 현지 벤더 전달용 수배 내역 (라운딩·카트·식사 등) |
| 저녁 명단표 | 저녁 | 저녁 식사 명단·배정 |
| 현지 정산표 | 정산 | 현지 비용 정산 (JPY 기준) |

- ExcelJS 스타일드 xlsx 다운로드 패턴 동일 적용.
- 태그코드 체계 `Y-MMDD-팀알파-번호` 유지 (확인 필요).

## 4. 포틴힐즈 특화 (SaiZen와 다른 점)

- 27홀 리조트(야마나미)가 아니라 **18홀 골프장** → 홀수·코스 상수 교체.
- 항공커버 탭은 제외 (4모듈만).
- 숙박 비중이 낮으면 수배서에 통합 처리.
- 출발지 코드(ICN / PUS / TAE) 그대로 사용.

## 5. Supabase (14hills 전용)

- 별도 프로젝트 `14hills` 생성 (서울 리전 권장).
- 초안 스키마: 첨부 `14hills_supabase_schema_초안.sql` → SaiZen 패턴에 맞춰 확정.
- 프론트엔드엔 `anon` 키만. `service_role` 키 노출 금지.
