14hills 운영 통합 시스템 BLUEPRINT

포틴힐즈 컨트리클럽(名古屋·나고야 / 18홀) 현장 운영 자동화 — SaiZen 야마나미 시스템 계열
회의용 구조 설명서
단일 HTML 앱 · Supabase 연동
산출물: 인쇄(PDF/종이)
한국 골프투어 송객을 받아 네임택 · 송영 · 현지 수배서 · 저녁 명단 · 현지 정산 산출물을 자동 생성하는 현장 운영 도구입니다. 엠클릭 예약 데이터(엑셀 2종)를 올리면 → 개인/팀 단위로 분석 → 월별 세션으로 정리 → 5개 모듈에서 인쇄물을 뽑습니다. 고객 소통은 한국어, 현장 산출물은 일본어 기조입니다.

1 전체 데이터 흐름 DATA FLOW

① 입력 (엑셀 2종)
엠클릭에서 받은 예약 데이터
  • 예약 리스트 — 예약현황(팀·상품·숙박·금액)
  • 일행별 예약 — 탑승자정보(개인·항공·여권·현지도착)
  • 여러 파일·여러 달 동시 등록 가능
→
② 분석
파싱·표준화·매칭
  • processed = 팀(예약) 단위
  • passengers = 개인 단위
  • 출발지 표준화(인천/부산/청주/대구)
  • 태그코드 자동 부여
→
③ 월별 세션
출발월(YYYY-MM)별 분리
  • 한 번에 올려도 달별로 자동 분리
  • 상단 바에서 월 전환
  • 클라우드 자동 저장/불러오기
→
④ 산출물 (5모듈)
현장용 인쇄물
  • 네임택 · 송영 · 수배서
  • 저녁 명단 · 현지 정산
  • 모두 인쇄(PDF/종이)로 출력

5 클라우드 · 세션 · 로그인 CLOUD & AUTH

📅 월별 세션 & 등록된 데이터
월별 자동 분리: 6~9월을 한 번에 올려도 출발월별로 각각 등록
상단 "등록된 데이터" 바: 등록된 월 목록(월·담당자·시각·팀/명) 표시, 클릭 전환
자동 저장: 분석을 마치면 클라우드에 자동 저장(전 월)
자동 불러오기: 다른 PC/새 사용자가 접속하면 최신 세션을 자동 로드
🔐 로그인 & 권한
로그인 필수: 미로그인 시 전체 화면 차단(사용 불가)
계정 발급: 로그인 화면에서 요청 → 마스터(관리자) 승인 후 발급
권한: 마스터/관리담당/일반 · 카드(모듈)별 접근 제어 가능
데이터 분리: 14hills 전용 Supabase 프로젝트(SaiZen과 별개)

6 기술 스택 & 운영 원칙 STACK

단일 HTML 바이브 코딩 Supabase(Publishable key) 엑셀 읽기 = SheetJS 출력 = 브라우저 인쇄(PDF/종이) 다운로드 파일명 영문 · 셀 내용 한·일 검증: node --check + jsdom 스모크
설계 원칙 — 고객 소통은 한국어, 현장 산출물은 일본어 기조. 코드가 항상 정답(문서와 어긋나면 코드 신뢰). 27홀(야마나미) → 18홀 단일 코스로 교체. 항공커버·배차/숙박 단독 탭은 미도입(수배서 한 장으로 통합).
